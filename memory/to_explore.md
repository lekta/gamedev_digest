На большом числе тел в многопотоке обгоняет дефолт Unity, на ткани проигрывает; WebGL заводится (1 поток). Мотивация Катто — Chaos в UE5 не подошёл. Апдейт 07-14: неделей позже вышел Jolt Physics 5.6 (13.07, Йоррит Роуве) — интерфейс compute-шейдеров на GPU (DX12/Vulkan/Metal), прядевая симуляция волос на GPU, glTF KHR_physics-моторы; используется в Horizon Forbidden West, Death Stranding 2, X4, Godot, GDevelop; к .NET — биндинги JoltPhysicsSharp (в Unity не «из коробки»). Собрать общий прогон обоих на боевой сцене: тысячи rigidbody, стабильность солвера, интеграция с DOTS/Jobs, накладные интеропа, что даёт GPU-путь. Апдейт 07-23: Godot 4.8 dev2 (21.07) штатно завёз Jolt 5.6 — сторонний движок стал дефолтом в мейнстрим-движке, что усиливает смысл очной ставки с дефолтной физикой Unity (и вопрос, почему у Unity Jolt только через обёртки). (07-07/07-14 апдейт 07-23 · box2d.org/posts/2026/06/announcing-box3d, github.com/Suvitruf/box3d-unity, github.com/jrouwe/JoltPhysics/releases)

- **Unity 6.6 deprecation `com.unity.serialization` — миграционный гид** — июньский CoreCLR-апдейт (16.06) официально пометил пакет deprecated. Рекомендованные альтернативы: System.Text.Json (не покрывает binary с offset-чтением для сейв-систем), Odin Serializer (платный, не всем подходит), MemoryPack (тянет NuGet for Unity). В discussions-треде уже видны живые жалобы. Собрать конкретные альтернативы под сценарии: сейв-системы, network messages, asset bundles, ScriptableObject-heavy редакторские расширения. Бонус — Burst встроен в ядро в 6.6 (embedded-копии при апгрейде сломаются), FEPM становится дефолтом, mimalloc как multi-threaded allocator. Появился отдельный «Path to CoreCLR, 2026: Upgrade Guide», прицельно описывающий шаги апгрейда. (06-29 апдейт · discussions.unity.com/t/coreclr-scripting-and-serialization-update-june-2026/1723299, discussions.unity.com/t/path-to-coreclr-2026-upgrade-guide/1714279)
