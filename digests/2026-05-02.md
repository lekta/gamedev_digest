# Сводка за 2026-05-02

## Главное

### Survios фактически закрылся: уволено большинство разработчиков
VR-студия Survios (Alien: Rogue Incursion, Creed: Rise to Glory), основанная в 2013, 1 мая лишилась почти всей команды разработчиков — сотрудники сообщают об этом в LinkedIn, один из них прямо пишет, что «Survios в её нынешнем виде по сути закрывается». Студия делала упор на лицензионные VR-проекты и в последний год пыталась выходить на «плоские» платформы, но это её не спасло. Для Unity-разработчика это очередной звонок про состояние VR-сегмента: Survios — не первая и явно не последняя VR-студия, которая в 2026 уходит на дно даже при «уже всё хорошо» в публичной риторике платформодержателей. Если строите проект под VR — стоит трезво смотреть на воронку финансирования и стабильность партнёров.
Источник: [Game Developer](https://www.gamedeveloper.com/business/report-vr-specialist-survios-has-laid-off-the-majority-of-its-development-team)

### Unity 6.8 окончательно убирает Mono — переезд на CoreCLR
Unity подтвердила в мартовском статус-апдейте, что в течение 2026 года будет последовательно выкатывать изменения скриптингового рантайма, а в Unity 6.8 Mono перестанет быть доступной опцией — основой C#-слоя движка станет CoreCLR. Команда обещает квартальные апдейты, влияющие на сериализацию, управление памятью и интеграцию с Entities. Для Unity-программистов это самая значимая структурная история года: меняется JIT, GC, модель загрузки сборок, а значит — поведение горячих путей, профилирование, Burst-взаимодействие и совместимость старых плагинов. Имеет смысл уже сейчас прогонять свои проекты на dev-ветках с CoreCLR, чтобы заранее ловить баги домашнего кода и сторонних пакетов, а не в момент LTS.
Источник: [Unity Discussions — CoreCLR, Scripting, and ECS Status Update](https://discussions.unity.com/t/coreclr-scripting-and-ecs-status-update-march-2026/1711852)

### .NET 11 Preview 3: Runtime Async выходит из «preview features»
Microsoft выпустила третий превью .NET 11 (релиз GA — ноябрь 2026). Главное: Runtime Async больше не требует флага EnablePreviewFeatures, теперь его поддерживают NativeAOT и ReadyToRun, появились новые JIT-оптимизации (устранение лишних bounds-check, сворачивание switch, интринсики Arm SVE2). В ASP.NET Core по умолчанию подключается Zstandard для compression/decompression, в SDK добавлен `dotnet run -e` для переменных окружения и директива `#:include` для file-based приложений. Прямо сейчас Unity на это не переедет (см. сюжет с CoreCLR выше), но для всего вне-юнитного C#-кода — внутренних тулз, билд-пайплайнов, серверного бэкенда — это уже пригодные к использованию вещи, особенно Runtime Async на NativeAOT.
Источник: [.NET Blog — .NET 11 Preview 3](https://devblogs.microsoft.com/dotnet/dotnet-11-preview-3/)

DLC The Dying World ещё выйдет, дальше — нет. [Game Developer](https://www.gamedeveloper.com/business/-the-company-as-a-whole-no-longer-exists-greedfall-developer-spiders-confirms-closure)

Студия отдельно открестилась от связи увольнений с недавним профсоюзным движением. [Game Developer](https://www.gamedeveloper.com/business/dead-by-daylight-studio-behaviour-interactive-confirms-layoffs)

- **Godot 4.6.2 — maintenance** — 1 апреля вышел багфикс-релиз 4.6 ветки, основная команда уже работает над 4.7. Полезно держать в курсе как опорную точку для миграционных разговоров. [Godot Engine](https://godotengine.org/article/godot-mobile-update-apr-2026/)

На фоне историй вроде Survios это любопытный контр-сигнал: Meta продолжает платить за инфраструктуру, тогда как сторонние VR-студии валятся.
