# Сводка за 3 июня 2026

## Главное

### Hell Clock: Cursed War — релиз отложен из-за окна Path of Exile 2
Бразильская Rogue Snail (Hell Clock — roguelike-ARPG на Unity, по мотивам Войны Канудос) должна была 2 июня выкатить консольную версию + дополнение Cursed War одним пакетом на ПК и консолях. За четыре дня до релиза студия публично объявила перенос «на несколько недель»: причина — Path of Exile 2 анонсировала свой апдейт на 29 мая, и стримеры/контентмейкеры, на которых маленький инди без бюджета на маркетинг опирается, всё это окно будут сидеть в PoE2. Кейс из разряда «план vs реальность»: студия делала ставку на синергию консольного запуска и DLC, но не угадала окно конкурента — и хладнокровно сдала позицию ради видимости, а не ради метрик первого дня. Для Unity-инди это редкий публичный пример, когда маркетинговая логика берёт верх над «выкатим как обещали», и заодно сигнал, что в 2026 году окно стримеров реально доминирует над производственной готовностью.
Источник: [Gematsu — Hell Clock Delayed](https://www.gematsu.com/2026/04/hell-clock-coming-to-ps5-xbox-series-on-june-2-alongside-cursed-war-expansion)

### Multi-agent VS Code и Autonomous Agent Mode — Build day 2 про инструменты разработчика
Если вчерашний keynote был про модели и платформу, то день 2 закрыл дыру по инструментам. GitHub Copilot multi-agent extension для VS Code уже доступен: вместо одного Copilot-агента, который последовательно прогоняет линтер → тесты → доку → security review, оркестратор раскидывает подзадачи параллельно. Параллельно анонсирован Autonomous Agent Mode для GitHub Copilot Enterprise (с июля): агент пишет, тестирует и коммитит ветку целиком в эфемерном Linux-контейнере (Agent Sandbox), к продовому репо доступа не получает — мерж только через ревьюера. Сегодня в 16:00 PT (BRK207) Microsoft показывает живую сессию, где Copilot-агенты в Visual Studio дебажат runtime-поведение, профилируют и доращивают тестовое покрытие. Для Unity-команд это первый внятный шанс параллелить рутину (генерация тестов под EditMode/PlayMode, документация asmdef-сборок, security review шейдерных пайплайнов) без болезненного per-step подтверждения.
Источник: [Visual Studio Blog — Build 2026](https://devblogs.microsoft.com/visualstudio/whats-coming-next-in-visual-studio-our-microsoft-build-2026-announcements/)

### .NET 11 Preview 4 — runtime-async выходит из preview-флага, MCP Server в SDK по умолчанию
Майский релиз .NET 11 Preview 4 (21 мая, прошёл мимо предыдущих выпусков — пора закрыть пробел) тащит две вещи, важные для Unity-горизонта CoreCLR 6.8. Первое: рантайм-библиотеки теперь скомпилированы с `runtime-async=on`, и для проектов на `net11.0+` он больше не требует `EnablePreviewFeatures=true` — то есть это уже не экспериментальный кусок, а дефолт. Стек-трейсы становятся чище: вместо лесов из машин-состояний компилятора в трейсе видны реальные async-методы (живой бонус для дебага многопоточных систем в Burst-обёртках). Второе: `mcpserver` project template теперь идёт в коробке с .NET SDK — `dotnet new mcpserver -o Foo` создаёт готовый каркас MCP-сервера на C#. Для Unity-разработчиков это значит, что собрать MCP-сервер для интеграции Editor-фич с любым агентом (Copilot, Claude Code, Junie) — теперь вопрос пяти минут, а не возни с отдельным NuGet. CoreCLR теперь default рантайм во всех таргетах MAUI, что косвенно подтверждает дорожную карту Unity на тот же CoreCLR.
Источник: [.NET Blog — Preview 4](https://devblogs.microsoft.com/dotnet/dotnet-11-preview-4/)
