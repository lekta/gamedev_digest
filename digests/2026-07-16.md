# Сводка за 16 июля 2026

## Главное

### Rider 2026.2 RC (15.07): IDE открывает свои «мозги» AI-агентам, Copilot встроили, всё стало быстрее
Вышел кандидат в релизы Rider 2026.2 — первый по-настоящему свежий движок по ядру за несколько тихих суток.
Для Unity-программиста главное вот что: внешний AI-агент (Claude Code, Codex) теперь может дёргать внутреннюю аналитику Rider как набор «навыков» — данные покрытия тестами, профайлер, инспекции ReSharper. То есть правки агента отбраковываются ещё до коммита, а не после. А на больших Unity-проектах с тяжёлым asmdef-графом заметны и чистые ускорения.
Ключевое: переключение веток в Roslyn-решениях в 2–3× быстрее, старт отладчика на Windows на ~2.8 c быстрее, память бэкенда ниже на 7–8%. Нативно встроили GitHub Copilot, добавили file-based шаблоны C# и Natvis-отладку на Linux/macOS. GA обещают в ближайшие недели.
Источник: [JetBrains Blog](https://blog.jetbrains.com/dotnet/2026/07/15/rider-2026-2-release-candidate-is-out/)

### Fields of Mistria (07.07): ферма-сим ушла с GameMaker на собственный движок и завела SteamOS
Догоняем на неделю: в патче v0.16.2 популярная Fields of Mistria (NPC Studio) переехала с GameMaker на внутренний рантайм Mistria SDK — прямо перед выходом из раннего доступа 5 августа.
Для нас это редкий публичный кейс «переросли стартовый движок → пишем свой», и по нему сразу видно цену вопроса. Сам переезд закрыл два застарелых бага (JSON Parse Error и Dragon Shrine Error) и включил поддержку интегрированной графики — но все моды под старый движок разом отвалились.
Кому интересен SteamOS как целевая платформа — полезна деталь: Linux поддержан только через Steam Linux Runtime 4.0 (steamrt4), запуск вне Steam студия не гарантирует. Для моддеров выложили отдельную документацию по новому движку.
Источник: [Fields of Mistria — Patch Notes v0.16.2](https://www.fieldsofmistria.com/post/new-engine-v0-16-2-patch-notes)

## По диагонали
- **ReSharper 2026.2 RC (15.07): отладчик приехал в VS Code и Cursor** — впервые за год в расширение ReSharper завезли отладчик (на том же движке, что в Rider) — это была самая запрашиваемая фича. Заодно первый шаг к агентам по протоколу ACP в Visual Studio (Junie в превью). [JetBrains](https://blog.jetbrains.com/dotnet/2026/07/13/rs-vsc-debugging/)
- **Спор о маркировке AI на Steam разгорелся заново** — поводом стала dark-fantasy ARPG Bahast, чей автор назвал генеративный AI «необходимостью для скорости»; коллеги-разработчики не поверили. По свежим данным, игры с меткой AI собирают примерно вдвое меньше отзывов, а продажи ниже на 40–60%. Для Unity-инди, что приглядываются к Inference Engine и AI-ассетам, — прямой сигнал: метка бьёт по видимости. [GamesRadar](https://www.gamesradar.com/games/devs-dont-buy-steam-disclosure-claiming-gen-ai-was-a-necessity-im-literally-unemployed-and-i-still-make-all-of-my-assets-by-hand/)
- **.NET: июльское сервисное обновление (14.07)** — вышли заплатки безопасности для .NET 8/9/10, закрыт набор уязвимостей (часть критические). По фичам ничего нового — просто повод обновить runtime и SDK. [.NET Blog](https://devblogs.microsoft.com/dotnet/dotnet-and-dotnet-framework-july-2026-servicing-updates/)
- **Palworld вышла из раннего доступа (1.0, 10.07)** — 40+ млн игроков, 94% положительных отзывов. В пару к Mistria — ещё один сюжет «взросления движка», только в обратную сторону: Pocketpair начинала на Unity и уехала на Unreal Engine 5. [Steam](https://store.steampowered.com/app/1623730/Palworld/)
