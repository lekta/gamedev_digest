# Сводка за 2026-05-12

## Главное

### Rider 2026.2 EAP 1 — Roslyn кратно быстрее на переключении веток
JetBrains 11 мая открыли Early Access для Rider 2026.2 с акцентом на боль больших солюшенов: переключение веток теперь в 2–3 раза быстрее в типовых сценариях, а в одном из их тестов (BenchmarkDotNet, ~25 проектов) сценарий ускорился с 8 минут до 5 секунд — ~100×. Работа сделана через батчинг и сокращение трафика между моделью проекта и Roslyn, заодно пофиксили зависания при передаче несуществующих файлов и ряд багов с `.editorconfig`, ссылками и target framework'ами. Для Unity-программистов отдельно переработали обработку asmdef-ссылок — Rider должен надёжнее обновлять модель проекта в больших солюшенах с разрезанием на сборки. Параллельно Azure Functions переезжает из отдельного плагина внутрь Rider, и допилен Aspire (file-based AppHosts, статус ресурсов в гуттере, debug в Docker).
Источник: [JetBrains .NET Tools Blog](https://blog.jetbrains.com/dotnet/2026/05/11/rider-2026-2-early-access-program/)

### ReSharper 2026.2 EAP — JetBrains тащит Junie и ACP Agent Registry в Visual Studio
Второй пост JetBrains того же 11 мая — целевой удар по конкурентам в самой VS. ReSharper 2026.2 EAP полностью посвящён одной фиче: интеграции Junie (агентного ассистента JetBrains) через открытый протокол ACP (Agent Client Protocol), с явной декларацией «no vendor lock-in». В первой превьюшке Junie уже умеет автономно писать и рефакторить код, гонять терминал и git, отвечать на проектные вопросы (доступ к solution-wide контексту, fine-grained пока нет). Дальше обещают ACP Agent Registry — каталог локальных/удалённых/корпоративных агентов, переключаемых через единый интерфейс. Это прямая ставка против Copilot Coding Agent и встроенного в VS Debugger Agent: JetBrains пытается превратить ReSharper в нейтральный «фасад» для любых AI-агентов внутри Visual Studio. Для шарпистов, которые сидят на VS из-за дебаггера или нативной интеграции с Windows-стеком, это первый реальный путь утащить агенты JetBrains/Junie без перехода в Rider.
Источник: [JetBrains .NET Tools Blog](https://blog.jetbrains.com/dotnet/2026/05/11/the-resharper-2026-2-eap-any-ai-agent-in-visual-studio/)

### Double Fine — 14-я Microsoft-студия с профсоюзом, единогласно
Авторы Psychonauts подали 7 мая петицию в NLRB через CWA, и за выходные история разлетелась по медиа: все 42 сотрудника Double Fine (full-time + part-time) подписались под объединением. Microsoft, как и в прошлых кейсах с ZeniMax/Activision, заявила о нейтралитете и не препятствует. Формально CWA подаёт параллельно request на voluntary recognition и election petition — резервный сценарий, если MS изменит позицию. Контекст: prior labor neutrality agreement Microsoft↔CWA истёк в 2025-м, его пока не переподписали. Это уже 14-я в Microsoft команда после серии волн 2023–2025, и кейс примечателен тем, что подаёт сигнал второму эшелону first-party-студий (Compulsion, Mojang, Ninja Theory) — а ещё попадает в текущий профсоюзный сюжет с MTG Arena в WotC, где в требованиях прописывали лимиты на генеративный AI.
Источник: [Kotaku](https://kotaku.com/psychonauts-maker-double-fine-just-became-the-14th-gaming-team-to-unionize-at-microsoft-2000694575)
