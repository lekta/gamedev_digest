### Yoshi and the Mysterious Book — Nintendo впервые публикует first-party игру на Unreal Engine 5
15 мая через лицензии в preload Switch 2 (SideFXLabs, Stylized Post Process, KawaiiPhysics — это плагины UE5) дата-майнеры подтвердили, что новый Yoshi (релиз 21 мая, эксклюзив Switch 2) сделан на Unreal Engine 5. До этого Nintendo держалась за собственные движки и UE4 (Yoshi's Crafted World, Princess Peach: Showtime!). Это очень громкий сигнал в копилку «крупные издатели уходят с Unity/in-house на UE5» — рядом с Marvel Snap (→Godot) и Subnautica 2 (Unity→UE5) у Unity накапливается неприятная серия флагманских уходов в первые недели мая. Для Unity-программиста — повод следить, как Unity ответит на Unite 2026 и что покажет в HDRP/URP против UE5-фич, ради которых студии перепрыгивают.
Источник: [Nintendo Life](https://www.nintendolife.com/news/2026/05/yoshi-and-the-mysterious-book-game-engine-seemingly-revealed) / [Wccftech](https://wccftech.com/nintendo-switch-2-doubted-over-unreal-engine-5-performance-yoshi-and-the-mysterious-book-test/)

### Claude Code 2.5 — большой релиз 15 мая с упором на плагины и фоновые сессии
Anthropic выкатили апдейт Claude Code: усилили работу с плагинами (`claude plugin disable` теперь отказывается выключать плагин, от которого зависят другие; `claude plugin enable` автоматически тянет транзитивные зависимости), показывают прогноз стоимости контекста в `/plugin` маркетплейсе, добавили `worktree.bgIsolation: "none"` для фоновых сессий без `EnterWorktree` (для репозиториев, где Git worktrees неудобны — например, Unity-проекты с большими бинарными ассетами и LFS), `PowerShell` теперь по умолчанию идёт с `-ExecutionPolicy Bypass`. С 14 мая Opus 4.7 стал моделью по умолчанию в Fast mode — это та же модель, что и обычный Opus 4.7, но в 2.5× быстрее и в 6× дороже по токенам ($30 за миллион input, $150 за output). Полезно как раз для итеративной работы по большому Unity-репу, где скорость отклика важнее цены за токен.
Источник: [Claude Code Changelog](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) / [Claude Code Docs — Fast mode](https://code.claude.com/docs/en/fast-mode)

### Microsoft открывает preview новых DirectX/PIX-инструментов
В мае публично стартует самая большая волна Windows-GPU-тулинга от Microsoft за всё время существования DirectX. В preview ушли: DirectX Dump Files (стандартизированный «крэш-дамп» GPU и состояния — раньше у каждого вендора было своё), `DebugBreak()` прямо в HLSL (брейкпоинт на уровне шейдера, без танцев с RenderDoc), Shader Explorer для PIX (compile-time perf insights, можно даже анализировать шейдеры для чужих GPU). Параллельно опубликован новый формат GPU-капчуров: Shader PDB кладётся прямо внутрь файла, в Проводнике видны превью капчуров. И — PIX API в публичном виде для C++, C# и Python. Для Unity HDRP/URP-разработчика это означает, что в течение нескольких месяцев должна появиться возможность отлаживать кастомные шейдеры почти как обычный код, и подключать PIX к Unity-билдам через C#-обёртку. Retail-доступ к Dump Files — около сентября.
Источник: [DirectX Developer Blog](https://devblogs.microsoft.com/directx/directx-bringing-console-level-developer-tools-to-windows/) / [Windows Developer Blog](https://blogs.windows.com/windowsdeveloper/2026/03/11/gdc-2026-announcing-new-tools-and-platform-updates-for-windows-pc-game-developers/)
