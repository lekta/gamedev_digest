# Сводка за 2 июня 2026

## Главное

### Microsoft Build 2026 keynote — Project Polaris отвязывает GitHub Copilot от GPT-4
Сегодня в Сан-Франциско открылась двухдневная Build. Главный технический анонс — Project Polaris, собственная in-house модель Microsoft, которая с августа заменит GPT-4 Turbo как дефолт в GitHub Copilot (автоматическая миграция, опциональный 3-месячный fallback на GPT-4 для команд, которые ещё не готовы). Архитектура — mixture-of-experts с под-модулями под отдельные языки и фреймворки; на Pro-тире обещают multi-file context до 100 000 строк и автономную генерацию тестов; крутится на собственных Maia AI-ускорителях в Azure. Для C#-разработчика это означает: к августу базовое поведение Copilot в Visual Studio / Rider-плагине / VS Code будет другим, бенчмарки на C# и Unity-репах придётся переснимать; плюс это политический сигнал — Microsoft публично режет пуповину к OpenAI ровно тогда, когда Anthropic с 15 июня переходит на metered billing для Agent SDK.
Источник: [ChatForest — Build 2026 Recap](https://chatforest.com/builders-log/microsoft-build-2026-recap-windows-agent-platform-project-polaris-copilot-workspace/)

### Copilot Workspace GA + Windows Agent Framework в open source
Параллельно с Polaris Microsoft выкатила в общую доступность Copilot Workspace (вышел из беты), open-sourced Windows Agent Framework и анонсировала Azure Agent Mesh. Workspace умеет рассуждать по всему репозиторию, предлагать мульти-файловые правки, гонять тесты и итерировать автономно — плюс Fleet Mode/Autopilot, где Copilot CLI работает по узко очерченным задачам без per-step подтверждения, и расширения для Jira/Datadog/ServiceNow прямо внутри сессии. Windows Agent Framework в open source — это база, на которой можно строить агентов поверх Windows Agent Runtime (об утечке писали 1 июня — runtime ships в Windows 11 24H2 через KB5039239, релиз 9 июня). Для Unity-стека ближайший практический эффект — Visual Studio 2026 Agent Skills получают auto-discovery SKILL.md прямо в чате, и теоретически Unity-команды смогут писать собственные скиллы под билд-пайплайн / тестирование EditMode/PlayMode без отдельного плагина.
Источник: [news.microsoft.com — Build 2026 Live](https://news.microsoft.com/build-2026-live-blog/)

### Bungie готовится к третьей волне увольнений, Destiny 2 закрывается 9 июня
Через неделю, 9 июня, выйдет финальный апдейт Destiny 2: Monument of Triumph — после него активная разработка прекращается, проекта на скамейке для команды D2 нет, Destiny 3 не зелёный. Bloomberg и техtimes подтверждают: третья волна layoffs идёт, размер не раскрыт, но «значительный». Контекст для бэкграунд-понимания: за фискальный 2025 Sony списала с Bungie $766M (импэйрмент — бухгалтерское обесценивание актива, когда компания на бумаге признаёт, что заплатила за студию заметно больше, чем она сейчас стоит): $565M в Q4 и $201M в Q2. Marathon остаётся единственным крупным проектом студии и пока не дотягивает до ожиданий по продажам. Для Unity/C#-разработчиков, работающих в live-service сегменте, кейс важен как иллюстрация: если игра-сервис теряет аудиторию, материнская компания обнуляет студию буквально по бух-балансу — это применимо и к unity-проектам с длинной поддержкой.
Источник: [Kotaku — Bungie Layoffs After Destiny 2](https://kotaku.com/destiny-2-final-update-bungie-layoffs-sony-marathon-2000698425)

## На разбор дальше
Главная живая тема — экономика Polaris vs Anthropic Agent SDK metered billing, который стартует 15 июня: имеет смысл прикинуть, что выгоднее для Unity-репы на 10–30k C#-файлов через две недели. Параллельно интересно посмотреть Glacier Next от IO Interactive (007 First Light) глазами Digital Foundry — релиз был 27 мая, DF deep-dive должен вот-вот выйти, фишки fully-RT GI и crowd-AI потенциально применимы в HDRP. Третий хвост — Unity Muse → Unity AI / Sentis → Inference Engine: переименование вроде уже катится, но публичных постмортемов от Unity о причинах отказа от first-party моделей пока не было. Если что-то из этого хочется поднять в приоритет на завтра — отметь любым способом.
