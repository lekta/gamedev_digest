# GameDev Digest

Все правила и память проекта живут в репозитории `lekta/gamedev_digest` на GitHub.
Этот скилл — только указатель: НЕ дублируй правила здесь, всегда читай их из репозитория.

## Порядок работы

1. **Прочитай правила** — `config.md` из репозитория:
   `https://raw.githubusercontent.com/lekta/gamedev_digest/master/config.md`
   Это единственный источник правил: тематика, структура, объём, язык, антиповторы, проверка фактов, первоисточники.

2. **Прочитай память** (тот же шаблон raw-ссылок, каталог `memory/`):
   - `memory/covered.md` — что уже освещено (антиповторы, 3-дневный кулдаун);
   - `memory/to_explore.md` — очередь тем на разбор;
   - `memory/sources.md` — реестр источников.

3. **Собери дайджест** строго по `config.md`. Дата выпуска — сегодняшняя.

4. **Сохрани результат:**
   - новый выпуск → `digests/YYYY-MM-DD.md`;
   - обнови `memory/covered.md` (окно 3 дня) и `memory/to_explore.md` по правилам из `config.md`.

5. **Куда писать:**
   - Если доступна локальная папка проекта GameDev Digest — сохрани файлы туда и сделай коммит.
   - Если работаешь в облаке с GitHub-коннектором — закоммить файлы прямо в репозиторий (ветка `master`, сообщение «Выпуск YYYY-MM-DD»).
   - Страница на GitHub Pages подтянет свежий выпуск сама — её трогать не нужно.
